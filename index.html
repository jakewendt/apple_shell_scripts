Apple Shell Scripts or ASS's

Apple Shell Scripts are super simple self contained commands that do what you ask. I have chosen to write these in "sh" which is one of several possible command line "languages" on the Mac OS X. These could potentially be aliases I suppose, but I've chosen to use them in file form. These scripts use Applescript's command line command "osascript" for execution. I don't know much about Applescripting. I just generally never used it, but I found that there were times when what I've needed would need to be via Applescript.

I watch podcasts and TV programs on my iMac and I usually sit in my LazyBoy with my powerbook surfing the web or writing some email. Multitasking. The phone rings. Now since I can't buy the hardware and software to install Front Row on my 2 year old ancient iMac, I have to get up and hit the space bar (rather than throw stuff at my keyboard in hopes of actually hitting it) or click pause or Mute. I suppose I could use a bluetooth keyboard or spend a couple hundred bucks on Apple Remote Desktop, but why?

Well now I don't and neither do you. Simply enable Remote Login from the System Preferences -> Internet & Network -> Sharing menu on your target computer, login via ssh and run iTunes.ToggleMute. Now sure, these scripts have their downside. They do take a second to compile, but this could be avoided by actually writing the script in the Script Editor, compiling and then saving that script in the appropriate format. As I said, I'm not much of an Applescript'r.

In addition to being a fruitful little utility, you could have a wee bit of fun with all of the computers that you have access. Play your favorite music instead of whatever it is that is currently being played.

Built-in Apple functions (all very obvious functionality)

  • open myDatafile (open's myDatafile with the default viewer)
  • exec myExec (run's myExec (Textedit, Terminal, ...)
  • osascript Applescript.scpt (run's Applescript.scpt)

Using Applescripts instead of open or exec will probably make it easier and you won't have to know the locations of the actual execs.
